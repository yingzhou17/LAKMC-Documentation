LKMC.Cascades module¶

Module to handle cascade events of fixed or probabilisticly selected energy during LKMC simulations using LBOMD.

Copyright Mark Wootton 2015

class LKMC.Cascades.CascadeRunner(params, refState)[source]¶

Bases: LKMC.ExternalEvents.ExternalEvent

For running collision events using MD during LKMC, one calls this from ExternalEvents.py

Parameters:

params : Input.InputParams

This object contains all parameters which are read from lkmcInput.IN by default.

refState : Lattice.Lattice

This object hold information about the reference state (perfect lattice) of a system.

Methods

getPKAEnergy()[source]¶

Set the PKA energy to be used in the collision cascade.

Returns:

energy: float

A randomly generated PKA energy (units: eV)
Raises:

IOError

If there is an error reading the probability file.

Notes

If one wishes to use a probabilisticly generated (i.e. non-fixed) PKA energy, this function will provide a value. These are generated from a user supplied input file describing the probability distribution, the path of which is specified by params.cascPKAProbFile.

getPKAVector()[source]¶

Creates a PKA vector, either randomly or using a pre-set value.

Returns:

pkaVector: numpy.ndarray

Cartesian three vector indicating a point on the unit sphere; used to describe the PKA direction.

Notes

If a random PKA vector is required it is chosen from a probability distribution evenly distributed over the unit sphere [R1].

References

[R1](1, 2) E.W. Weisstein. Sphere Point Picking. From MathWorld-A Wolfram Web Resource. http://mathworld.wolfram.com/SpherePointPicking.html
run(latticeIn, step=None, clock=None)[source]¶

Calls all of the methods to do one complete cascade simulation.

Parameters:

latticeIn : Lattice.Lattice

System lattice to run a cascade in.

step : int, optional

Current KMC step, to allow changing into the appropriate working directory.

clock : float, optional

System time of KMC simulation, used to record time of last cascade event, facilitating simulation resume.

Notes

The workflow is as follows:
  • Pre-event thermalisation
  • Generating PKA attributes
  • Adding the PKA
  • Simulating the cascade
  • Post-cascade minimisation
