LKMC.Lattice module¶

This module contains the Lattice class and associated methods.

class LKMC.Lattice.Lattice(NAtoms, storeEnergies=False)[source]¶

Bases: object

The Lattice class holds information about the state of a system.

Parameters:

NAtoms : integer

The number of atoms to initialise the system with (note this can be set to 0 if you want to add atoms one at a time (eg. using the Lattice.addAtom method).

storeEnergies : bool, optional

Set this to ‘True’ if you want to store potential (Lattice.PE) and kinetic (Lattice.KE) energies for each atom. If set to ‘False’ this will only store the total energy of the Lattice.

Attributes

NAtoms (integer) The number of atoms in the system.
cellDims (ndarray) Dimensions of the simulations cell.
specieList (ndarray) Array containing two character species symbols found in the the Lattice.
specieCount (ndarray) List of counters indicating the number of atoms of each species (from the Lattice.specieList).
specie (ndarray) Array of integers, indicating the species of each atom in the Lattice (i.e. the index of their symbol in the Lattice.specieList).
pos (ndarray) The positions of the atoms in the Lattice. Positions are stored like this [xpos_0, ypos_0, zpos_0, xpos_1, ypos_1, ...]. The length of the array is 3 x NAtoms.
charge (ndarray) The charges of the atoms.
force (ndarray) The forces acting on the atoms. The structure is the same as the Lattice.pos array.
totalEnergy (float) The total energy of the system.

calcForce(MDPath=None, defectFlag=0, correctTE=False, includeAtoms=array([], dtype=int32), mainIndexes=array([], dtype=int32))[source]¶

Calculate forces acting on the Lattice.

Parameters:

correctTE : bool, optional

Compute the correct total energy, i.e. ignore defectFlag.

defectFlag : {0, 1, 2}, optional

Calculate forces on defect sub-system.

  • Set to 0 to do a normal force calculation (default)
  • Set to 1 to zero the forces on atoms within the sub-system
  • Set to 2 to zero the forces on atoms outside the sub-system

mainIndexes : ndarray, optional

Specify the main atoms to use in the defectFlag calculation. These atoms make up the defect sub-system. If not specified it defaults to Lattice.defectSubSystem.

includeAtoms : ndarray, optional

Specify the atoms to include in the force calculation to make sure the energies and forces on the main atoms are correct. If not specified then these atoms are determined by building a sub-lattice (see Lattice.buildSubLattice) around the main atoms using the parameter: subLatticeIncludeRadius.

MDPath : str

Deprecated! This setting will likely be removed in a future release!
Returns:

status : integer

0 means success, anything else is a failure.

getMinSepList(inputPos, atomList, maxSpacing=10.0, atomListSize=None)[source]¶

Returns minimum separation between inputPos and an atom from atomList.

Parameters:

inputPos : np.ndarray[3]

The position you want to check.

atomList : np.ndarray

The indices of the atoms from the Lattice you want to check against.

maxSpacing : float

The maximum spacing between atoms in the lattice. This is used for domain decomposition; if set too small could result in there being no points near inputPos. You can check if this has happened since minSep will be larger than maxSpacing.

atomListSize : int, optional

The size of atomList. If this parameter is not set then we use the full array (i.e. this parameter can be used if wish to only consider the first atomListSize elements ofe the atomList array).
Returns:

minSep : float or None

The minimum separation found between inputPos and an atom from atomList. If no atom was found within maxSpacing of the inputPos then retun None.

minSepIndex : int or None

The index of the site within atomList that was closest to inputPos or None if no atom was found within maxSpacing. Note this is not the index within the Lattice but is the index within atomList, i.e. 0 <= index < len(atomList).
Raises:

ValueError

If atomListSize is specified and it is out of range (bigger than len(atomList) or negative).
